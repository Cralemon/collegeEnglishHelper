# 软件开发方案

**软件功能及期望**：
面向大学生。总体功能：提供英语翻译题目的练习，并在翻译题目练习的过程中逐步引导学生建立词汇和语法等的更高level。
设计哲学：运作单位为每一道翻译题。软件提供卡片式的翻译类题目（单句/段落可切换），用户输入自己的答案后提交，软件分析用户所给答案的优缺点，并针对性的进行提升点的建议。通过多轮类似输入与反馈，建立用户对提升点的能力提高。

**需求技术栈**：react 或 原生 HTML，若需打包应用，考虑electron/tauri。

**具体细节**：high-level prompt: 在需要确保robustness的地方使用程序编写以确保稳定性，在需要学习方面针对性内容的地方使用LLM以提供丰富性并确保准确性。LLM接口由用户提供。
应当包含如下页面：
- **首页（翻译练习）**：卡片式层叠翻卡设计。题目及作答、提交按钮为正面，AI反馈为反面。给出AI反馈后即可“划卡”进入下一题。AI反馈包含三个维度，以及一句简短总结。给出打分系统（百分制）。缓存做过的题，并允许建立和切换自己的题集。提供当前卡牌集合的打乱功能。
- **回顾（改进点的收录）**：基础的统计数据：刷题数量、平均分、各分段的题目数量、相关图表等。允许在不创建题集的情况下将某一题目集合（如 50分以下的作答历史等）推入首页再次练习。  维护并展示三个维度的改进点：语法、词汇、句型。 均包含刷题过程中AI给出的反馈内容。
- **个人/设置**：提供用户信息的管理，此处填写的信息将用于LLM更针对性的给出学习指引。设置项：昵称、学年段、预估词汇量、翻译练习偏好设置：整段/单句、中译英/英译中、主题切换、[待构想]。提供一个接口，指引用户进行简单的测试，以评估其英语水平。维护一个personal portrait，以提高个性化程度。

**软件仅供个人使用，因此无需在隐私保护和数据安全方面进行过多考虑。**

软件使用的提示词设计需要确保反馈的准确性以及格式符合性。使LLM输出特定的格式以确保能够被正确解析并收录。

每个模块均应当确保可维护性和可拓展性。

**UI设计**：基础UI风格采用Claude。允许用户在有限个不同主题风格之间切换。

**交互设计**：优雅动画。
